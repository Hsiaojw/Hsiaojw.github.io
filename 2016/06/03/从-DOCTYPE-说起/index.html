从 DOCTYPE 说起

发表于 2016-06-03   |   分类于 笔记   |  

发现 DOCTYPE 可说的部分还是有很多的，深入了解一番。

DOCTYPE 的作用

在这次专门了解 DOCTYPE 的作用之前，我一直想当然的认为其作用如下：

  1. 使开发者了解正在使用的是什么文档标准（可能是 XHTML 或者某个版本的 HTML）。
  2. 通过 DOCTYPE 告诉浏览器正在使用什么文档标准，从而浏览器可以用符合这个文档标准的方式进行解析。

但是通过这次较为深入的了解后，发现其实跟我过去想的有很大差别。

什么是 DOCTYPE

首先肯定是查看官方资料的。从 W3.org 中，可以看到对 DOCTYPE 的解释如下：

DOCTYPEs are required for legacy reasons. When omitted, browsers tend to use a different rendering mode that is incompatible with some specifications. Including the DOCTYPE in a document ensures that the browser makes a best-effort attempt at following the relevant specifications.

简而言之，不添加 DOCTYPE 声明可能会造成浏览器用旧的方式渲染网页。 添加正确的 DOCTYPE 声明，可以确保浏览器使用标准模式进行渲染，除此之外，并没有其它特定功能。在进行网页构造时，第一件事就是先写下 <!DOCTYPE html> 。

实际上这就是本文的结论了。

渲染模式

我们知道，一个网页的结构与外观由两种标准化的语言的组合所表述：HTML 和 CSS。然而，多数旧的网页浏览器并没有完全实施这些语言的定义，或者浏览器在这些语言的定义完善之前就已经被开发出来了。这导致的结果是，过去网页的构造依赖于早期浏览器不完全或不正确的解析，并且只有在被这样的浏览器处理时才会如开发者本身所设想的那样进行渲染。

后来，随着标准的建立，主流浏览器对标准化的 HTML 和 CSS 的支持得到了显著提高。然而，早期那些已经被开发出来的非标准化网页仍然大量的存在于互联网中，同时许多新的网页也继续以旧的方式被建造出来。因此，浏览器开发者不得不在提高对标准化 HTML 和 CSS 支持的前提下，考虑对非标准化的网页进行向后兼容。

为了同时兼顾依据 Web 标准和 90 年代末流行的传统做法开发出来的网页，现代的网页浏览器一般具有多种渲染模式。这些渲染模式包括：

  • Standards Mode 标准模式
  • Almost Standards Mode 近乎标准模式
  • Quirks Mode 怪异模式

对于 Quirks Mode （怪异模式），维基百科上的解释如下：

Quirks Mode （怪异模式）是指计算机领域中，一些网页浏览器为了维持对较旧的网页设计的向后兼容性，而使用的一种技术，有别于严格遵守万维网联盟（W3C）与互联网工程任务组（IETF）标准而设计的 Standards Mode 标准模式。

Standards Mode （标准模式）下，页面严格按照 HTML 与 CSS 的定义渲染；Almost Mode （近乎标准模式），与标准模式稍有区别；Quirks Mode （怪异模式），尝试模拟更旧的浏览器的行为。

触发不同的渲染模式

通常，浏览器给予页面中文件类型描述的存在以决定采用哪种渲染模式，也就是根据页面的 DOCTYPE 声明决定。概括来说，如果使用一个完整的 DOCTYPE 则浏览器将会采用 Standards Mode （标准模式），如果缺失、不完整则浏览器将会采用 Quirks Mode （怪异模式）。

具体的差异及触发不同渲染模式的情况可以在这个页面进行了解。

模式验证

在绝大多数浏览器，DOM 的扩展 document.compatMode 指示当前页面的渲染模式。document.compatMode 包含值如下：

  • CSS1Compat：Standards Mode 和 Almost Mode
  • BackCompat：Quirks Mode

HTML 和 XHTML 的区别

上结论：XHTML 是 XML 版的 HTML，一个语法严格的 HTML。这就是说，XHTML 是 XML，但 HTML 不是 XML。

首先，我们需要了解 XML 是什么——XML 是被设计用来传输和存储数据的可扩展标记语言。人们可以很容易的理解它，机器也可以很容易的解析它。许多 Internet 语言是通过 XML 创建的。

早期发展中，HTML 的标准不是很规范，语法较为松散。相比之下，XML 则要严谨规范得多。因此，为了规范 HTML ，W3C 结合 XML 的规则制定了 XHTML。还定义了一个新的 MIME type，Content-type: application/xml+xhtml 。这样，浏览器会对这个 MIME Type 强制查错，即一旦页面含有错误就提示。

两者主要区别如下：

  • XHTML 所有元素必须被正确的嵌套
  • XHTML 所有元素必须闭合
  • XHTML 所有元素必须小写
  • XHTML 所有元素必须有一个根元素
